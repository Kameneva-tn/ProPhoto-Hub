# Сайт фотостудії PRO PHOTO studio hub

Верстка повторює макет Figma (`node-id=42:821`, файл `STUDIO_HIGHLIGHTS`) — сітка, тексти,
кольори, фото/іконки, шрифт. Стек свідомо простий, без React/білдера:

```
index.html          — всі 5 екранів як секції з якорями (#home #halls #booking #equipment #contacts)
css/style.css        — стилі; CSS-змінні для кольорів/розмірів (--c-*, --pill-bg, --container)
js/main.js           — вся логіка: авто-зум фону, модалка, зали, сітка бронювання, відправка форм
js/halls-data.js     — контент і ціни по залах, окремо від логіки (HALLS, NIGHT_MULTIPLIER)
public/assets/       — фото, іконки, лого з макета
public/fonts/        — KyivType Sans, реальні .woff2 (7 нарізок ваги: Thin…Black)
server/server.js     — Express-бекенд: сітка зайнятості + бронювання + Telegram
server/bookings.json — рантайм-"база" бронювань (у .gitignore — не в репозиторії)
```

## Що на сайті

| Секція | Що там |
|---|---|
| Головна | Фон "дихає" (CSS `@keyframes`, +30% масштаб за 15с і назад), лого на прозорому PNG, кнопка "Забронювати", кругла кнопка "передзвоніть мені" |
| Зали | 3 зали (Циклорама/Подкаст зала/Гримерна), клік по фото відкриває картку з розмірами, фічами й ціною оренди |
| Бронювання | Сітка "дні (горизонталь) × години (вертикаль)" з ціною в кожній клітинці; **мультивибір** — можна обрати кілька годин одразу; денна/нічна ставка (+50% до 10:00 і після 20:00) |
| Обладнання | Картки техніки по залах (спалахи Godox, постійне світло Ulanzi/AC Prof) з фото й характеристиками |
| Контакти | Адреса, телефон, інстаграм, режим роботи |
| Модалка | "Передзвоніть мені" — ім'я+телефон → Telegram |

## Бронювання: як це працює під капотом

1. Клієнт обирає зал (`.hall-tab`) і одну або кілька клітинок у сітці (`js/main.js` → `booking()`).
   Вибір тримається в `Map` (`state.selectedSlots`), переживає гортання по тижнях.
2. Ціна за годину рахується з `HALLS[hall].rate` (базова денна ставка) і `NIGHT_MULTIPLIER`
   (`js/halls-data.js`) — одна константа, а не окремі числа в кожному місці.
3. `POST /api/book` приймає масив `slots` і перевіряє конфлікти по **всіх** годинах одразу,
   до запису — щоб не вийшло наполовину заброньованого запиту, якщо один слот зайняли
   на секунду раніше.
4. Заявка одразу летить у Telegram-чат менеджерки (`sendTelegramMessage` в `server.js`).

## Запуск локально

```bash
# 1. Бекенд
cd server
npm install
cp .env.example .env      # вписати TELEGRAM_BOT_TOKEN і TELEGRAM_CHAT_ID
npm start                 # http://localhost:3000 — віддає і фронт, і API

# або просто відкрити index.html без бекенду —
# сайт працює, форми покажуть повідомлення "зателефонуйте нам напряму"
```

Як отримати `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID` — покроково в `server/.env.example`.

## Шрифт

KyivType Sans підключений по-справжньому (`public/fonts/*.woff2`, `@font-face` в
`style.css`, 7 нарізок ваги Thin→Black). Ліцензійний файл наданий власницею сайту,
у репозиторій він потрапляє разом з рештою `public/` — якщо цей репозиторій публічний,
варто перевірити умови ліцензії шрифту перед тим, як робити репо відкритим.

## Що ще можна автоматизувати кодом

1. **Синхронізація сітки бронювання з Google Calendar** — замість `bookings.json`
   тримати зайнятість у Google Calendar API, щоб менеджерка бачила бронювання
   у звичному календарі без ручного дублювання.
2. **Telegram-бот з кнопками підтвердження** — зараз повідомлення інформативне;
   можна додати inline-кнопки "Підтвердити / Відхилити", які одразу міняють статус.
3. **CRM-мінімум**: `bookings.json` → SQLite/Supabase — історія клієнтів,
   ремаркетинг-списки, без окремого CRM. Важливо і для GitHub-гігієни:
   файлова "база" з іменами/телефонами не повинна лежати в git.
4. **Автоматичний OG-image** для посилань на конкретний зал/подію в соцмережах.
5. **A/B копірайтингу кнопки "Забронювати"** через просту split-систему + GA4/Meta Pixel.
